# 面試自我介紹稿

您好，我是謝秀玟，目前是一位軟體工程師，主要經驗集中在工具平台開發、嵌入式開發環境、視覺化應用，以及 CI/CD 自動化。

我在晶心科技擔任高級工程師，負責過開發工具中的診斷視圖、暫存器視覺化、多核心除錯與效能分析相關功能。這段經驗讓我比較熟悉複雜系統中的問題拆解，尤其是 legacy 環境相容性、非同步流程、測試穩定性，以及工具架構可維護性的議題。

我自己比較明顯的工作特質，是會先找出問題根因，再判斷應該用修補、重構，還是工具化的方式解決。例如我曾經處理 GUI 自動化測試在 CI/CD pipeline 中頻繁失敗的問題。當時錯誤表面上是 `component not found`，但我透過 runlog、執行時序與環境差異追查後，確認根因是 Daemon 模式下 RMI 延遲與 Swing/AWT render 時序不同步。最後我不是單純調整測試腳本，而是把自動化測試改成在 Jenkins Agent 本地執行，並補強等待機制，讓 flaky test 對 pipeline 的阻塞大幅降低。

另一個比較代表性的專案，是 AutoOpTune 視圖開發。這個功能需要用 JCEF Web View 呈現分析結果，但產品又必須支援 CentOS 7.4 這類已停止維護的舊環境。由於 GLIBC 版本過舊，沒有現成 binary 可以直接使用，我後來建立 Docker 編譯環境，控制 devtoolset、yum repo 與 GLIBC 相容性，手動編譯 JCEF 126 與相關相依元件，最後讓功能在舊環境中也能如期上線。這個經驗讓我對環境相容性、相依管理與產品承諾之間的取捨有更深的理解。

除了產品功能開發，我也很重視把重複性的人工流程工具化。像是我曾經把 SPA 規格文件中數百個 CSR 與 bitfield 定義，自動轉換成 JSON 描述檔、GDB target description 與 C header，將原本大約 5 天的手工作業縮短到數分鐘。這類工作對我來說不只是節省時間，也能降低版本更新時的人為錯誤，讓後續維護更穩定。

近期我也把這樣的工具化思維延伸到 AI Agent 與全端 PWA 開發。我做了一個 Personal Travel Dashboard side project，使用 React、Vite、Firebase、Dexie 與 IndexedDB 實作 offline-first 的旅遊管理系統，並整合多模型 API、Function Calling、PDF/Office 文件解析與座標自動補齊。這個專案讓 AI 不只是回答問題，而是能透過工具呼叫實際建立行程、操作頁面與補齊資料。

整體來說，我的優勢是能在複雜系統中找出問題核心，並把解法落成可維護、可驗證、可自動化的工程產出。我希望未來能持續投入工具開發、平台工程、嵌入式除錯或自動化相關工作，加入重視技術深度與交付效率的團隊，處理有挑戰性的系統問題。
